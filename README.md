# Предельная ценность распределительной информации для правила процентной ставки

Репозиторий содержит код и материалы к работе о ценности распределительной информации при выборе
правила процентной ставки в линеаризованной HANK/SSJ-модели с неполной наблюдаемостью.

## Идея

Центральный банк наблюдает не истинное состояние экономики, а шумные агрегатные и
распределительные сигналы. Работа сравнивает несколько информационных режимов и оценивает,
снижает ли распределительная информация потери стабилизации сверх агрегатной оценки состояния,
построенной по фильтру Калмана.

## Постановка

HANK/SSJ-отклики используются для построения локальной модели в пространстве состояний. Для
каждого информационного режима заново настраивается однотипное линейное правило процентной
ставки. Сравнение проводится по одной и той же квадратичной функции потерь стабилизации.

Основная величина сравнения: $\Delta J_{\mathrm{dist}} = J^\star_{\mathrm{agg}} - J^\star_{\mathrm{agg+dist}}\]$

Положительное значение означает снижение потерь при добавлении распределительной информации к
агрегатной оценке состояния.

## Основной результат

Наиболее устойчивый выигрыш даёт агрегатная оценка состояния по фильтру Калмана. Добавление
полного набора распределительных статистик сверх этой базы в основной спецификации не даёт
статистически устойчивого снижения потерь. Распределительная информация имеет условную, а не
универсальную ценность: она полезна тогда, когда помогает уточнить состояние, связанное с будущей
реакцией экономики на изменение ставки.

## Структура репозитория

- `hank/` - HANK-ядро и распределительные статистики.
- `hank_ssj/` - SSJ-отклики, наблюдения и локальная модель в пространстве состояний.
- `policy/` - правила процентной ставки, функция потерь и линейно-квадратичный ориентир.
- `experiments/final/` - финальные точки входа для воспроизведения работы.
- `experiments/archive/` - исторические и вспомогательные расчётные скрипты.
- `article/` - текст работы, таблицы и рисунки.
- `docs/` - технические заметки, воспроизводимость и архивные описания расчётов.
- `config/` - параметры расчётов.

## Основное воспроизведение

Полный путь воспроизведения собран в одном скрипте:

```bash
bash scripts/run_article_pipeline.sh
```

Ключевые этапы можно запускать отдельно:

```bash
python3 scripts/run_hank.py --output-dir outputs/hank_core
python3 experiments/final/00_audit_hank_core.py --hank-core-dir outputs/hank_core
python3 experiments/final/01_validate_ssj.py
python3 experiments/final/03_estimate_information_value.py
python3 experiments/final/04_mechanism_checks.py
python3 experiments/final/05_lqg_benchmark.py
python3 experiments/final/06_feedback_rate_check.py
```

Подробная схема воспроизводимости приведена в `docs/reproducibility.md`.

## Терминология

Единый словарь проекта находится в `docs/terminology.md`. 

- распределительные статистики - функции от распределения домохозяйств
- распределительные сигналы - шумные наблюдения или оценки этих статистик
- информационный режим - способ наблюдения и сжатия информации регулятором
- агрегатная оценка состояния - оценка агрегатного состояния по фильтру Калмана
- предельная ценность информации - снижение оптимизированных потерь при переходе между
  информационными режимами
